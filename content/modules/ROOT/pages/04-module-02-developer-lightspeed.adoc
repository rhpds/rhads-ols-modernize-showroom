== Exercise 2: Run an initial static code analysis

Before making changes, you'll use Developer Lightspeed MTA extension to analyze the legacy code and identify specific modernization opportunities.

In this exercise the Developer Lightspeed for MTA extension will run a static code analysis against all the source code in the cloned repository, including the custom ACME specific company policy code rules. This provides an efficient starting point focused on identifying migration issues that were highlighted during the risk analysis in the last module.

Later on the analysis results will be used with the LLM for generating code fixes.

=== Prerequisites
* Dev Spaces workspace running with Developer Lightspeed extension configured
* Repository **kai-coolstore** successfully cloned

=== Steps

. Start the Analysis Server
+
* Navigate to the **MTA Analysis View**
* In the center of the screen make sure the profile selected is the **{user} coolstore profile**
* At the top right of the screen, ensure the __Server Status__ is **Started**
** Click the **Start** button if it isn't and wait till the Server Status is **Started**

. Invoke Developer Lightspeed analysis:
+
* In the right side of the screen click on the **Run Analysis** button.

. Review the AI analysis results:
+
Developer Lightspeed will identify:
+
* Deprecated APIs that need updating
* Configuration patterns that should be modernized
* Container-readiness issues
* OpenShift-specific optimizations

. Examine specific recommendations:
+
Click on each finding to see:
+
* Description of the issue
* Why it matters for containerization
* Suggested code changes
* Impact on application behavior

. Take note of high-priority items:
+
Focus on changes marked as:
+
* **Critical**: Must fix for successful containerization
* **Recommended**: Should fix for best practices
* **Optional**: Nice-to-have improvements

=== Verify

Confirm the analysis provides actionable insights:

✓ Developer Lightspeed identified modernization opportunities
✓ Findings are categorized by priority
✓ Specific code locations are highlighted
✓ Remediation suggestions are provided

== Exercise 3: Generate modernized code using AI

Now you'll use Developer Lightspeed to automatically generate code changes based on the analysis. This is where AI acceleration becomes real for ACME.

=== Steps

. Select a modernization task from the analysis results:
+
Start with a critical or recommended finding that has clear remediation guidance.

. Invoke AI code generation:
+
* Click on the finding
* Select **Apply AI Fix** or **Generate Solution**
* Developer Lightspeed will generate the modernized code

. Review the proposed changes:
+
Developer Lightspeed will show you:
+
* Original code (before)
* Generated code (after)
* Explanation of what changed and why
* Impact assessment

. Examine the generated code quality:
+
Check that the AI-generated code:
+
* Follows Java best practices
* Maintains application functionality
* Uses modern APIs and patterns
* Includes appropriate error handling
* Contains helpful comments

. Accept the AI-generated changes:
+
If the code looks correct:
+
* Click **Accept Changes**
* The file will be updated with the modernized code

. Verify the changes compile:
+
[source,bash]
----
# Build the application to confirm no syntax errors
./mvnw clean compile
----

. Repeat for additional findings:
+
Select another modernization opportunity and repeat the AI generation process. This builds confidence in the AI's capabilities.

== Exercise 4: Review and validate AI-generated changes

Before committing AI-generated code to ACME's repository, you need to thoroughly review and validate the changes. This ensures quality control in the modernization process.

=== Steps

. Open the Git diff view to see all changes:
+
[source,bash]
----
# View all modified files
git status

# See detailed changes
git diff
----

. Review each modified file:
+
For each file with AI-generated changes:
+
* Understand what changed and why
* Verify the logic is correct
* Confirm it aligns with MTA recommendations
* Check for any unintended side effects

. Run comprehensive tests:
+
[source,bash]
----
# Run full test suite
./mvnw verify

# Check code coverage if available
./mvnw test jacoco:report
----

. Perform manual testing:
+
* Start the application locally
* Test key functionality
* Verify containerization-related changes
* Confirm configuration is correct

. Commit validated changes to version control:
+
[source,bash]
----
# Stage the changes
git add .

# Commit with descriptive message
git commit -m "Modernize application for containerization using AI assistance

- Updated deprecated APIs to modern equivalents
- Applied containerization best practices
- Improved configuration for cloud deployment
- Generated with Red Hat Developer Lightspeed"

# Push to Gitea
git push origin main
----

=== Verify

Confirm your changes are properly validated:

[source,bash]
----
# Verify commit was successful
git log -1

# Confirm push to remote
git remote -v
----

✓ All AI-generated changes reviewed and understood
✓ Tests pass successfully
✓ Changes committed to version control
✓ Code is ready for next phase

== Exercise 5: Expand migration with Agent Mode

For ACME's large-scale migration, you need to modernize entire projects, not just individual files. Developer Lightspeed's Agent Mode enables AI to work autonomously across the codebase.

=== Steps

. Activate Developer Lightspeed Agent Mode:
+
* Open Developer Lightspeed panel
* Click **Agent Mode** toggle
* Confirm activation

. Define the migration scope:
+
Provide instructions to the AI agent:
+
[source,text]
----
Modernize this Spring application for containerization on OpenShift:
1. Update all deprecated Spring Boot APIs to latest versions
2. Apply container-ready configuration patterns
3. Add health check endpoints
4. Externalize configuration for cloud environments
5. Update dependencies to container-compatible versions
----

. Let the Agent work:
+
Developer Lightspeed Agent Mode will:
+
* Analyze the entire project structure
* Identify all files needing changes
* Generate modernization code across multiple files
* Apply changes systematically
* Provide progress updates

. Monitor the Agent's progress:
+
Watch as the Agent:
+
* Scans project files
* Makes decisions about changes
* Applies transformations
* Validates its own work

+
Once the Agent finishes:

[source,bash]

== Troubleshooting

**Issue**: Developer Lightspeed extension not responding
**Solution**: Verify AI model endpoint is accessible:
[source,bash]
----
curl -k {litellm_api_endpoint}/health
----
If endpoint is down, check with workshop instructor. If accessible, try restarting the extension or workspace.

**Issue**: AI generates incorrect code that breaks tests
**Solution**: Review the AI-generated changes carefully. Use git to revert specific changes:
[source,bash]
----
# Revert to previous version
git checkout -- <file-name>

# Try a more specific prompt to the AI
# Be explicit about constraints and requirements
----

**Issue**: Agent Mode makes too many changes at once
**Solution**: Provide more specific, constrained instructions:
[source,text]
----
Focus only on updating Spring Boot dependencies in pom.xml
Do not modify application logic or configuration files yet
----

**Issue**: Build fails after AI modifications
**Solution**: Check compiler errors and resolve step-by-step:
[source,bash]
----
# See specific compilation errors
./mvnw compile 2>&1 | grep error

# Fix errors one file at a time
# Ask Developer Lightspeed to fix specific compilation errors
----

**Issue**: Cannot push to Gitea repository
**Solution**: Verify Git credentials and remote configuration:
[source,bash]
----
# Check remote URL
git remote -v

# Verify Gitea credentials
# Username: {user}
# Password: {password}

# Push with explicit credentials if needed
git push https://{user}:{password}@<gitea-host>/<repo>.git main
----
